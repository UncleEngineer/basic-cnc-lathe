1. โครงสร้างโปรแกรม Heidenhain

Heidenhain ใช้ภาษา "Conversational" (ภาษาธรรมดา) แทน G-code มาตรฐาน — ทำให้อ่านง่ายกว่า Fanuc/Siemens มาก แต่ก็รองรับ ISO G-code ได้เช่นกัน

1
BEGIN PGM / BLK FORM — ประกาศโปรแกรม & Workpiece
บรรทัด 0–2 เสมอ
2
TOOL DEF / TOOL CALL — กำหนดและเรียก Tool
บรรทัด 3–4
3
Start-up — ยก Z, เปิด Spindle, เปิด Coolant
บรรทัด 5–7 (ตายตัว)
4
Machining — เคลื่อนที่ / เจาะ / กัด
บรรทัด 10+ (นับทีละ 2)
5
End — ยก Z, M2, END PGM
บรรทัดสุดท้าย
จุดเด่น Heidenhain vs Fanuc: ใช้คำว่า L แทน G01, CC+C แทน G02/G03, CYCL DEF แทน Canned Cycle — อ่านง่ายกว่ามาก

2. ส่วน Header — บรรทัด 0 ถึง 2

Header ตายตัว ทุกไฟล์
0BEGIN PGM 7227 MM
1BLK FORM 0.1 Z X+0 Y+0 Z-50
2BLK FORM 0.2 X+100 Y+100 Z+0
คำสั่งความหมายหมายเหตุ
BEGIN PGM [N] MMเริ่มโปรแกรม N หน่วย mmใช้ INCH แทน MM ได้ถ้าเป็นนิ้ว
BLK FORM 0.1 Z X.. Y.. Z..มุมล่าง-ซ้าย ของ Workpiece (สำหรับ simulation)Z axis ของ Spindle คือ Z
BLK FORM 0.2 X.. Y.. Z..มุมบน-ขวา ของ WorkpieceZ+0 = ผิวหน้าชิ้นงาน
END PGM [N] MMจบโปรแกรมN ต้องตรงกับ BEGIN

3. กำหนดและเรียกใช้ Tool

TOOL DEF & TOOL CALL บรรทัด 3–4
3TOOL DEF 1 L+0 R+3 ← นิยาม Tool 1: ความยาว +0, รัศมี +3mm
4TOOL CALL 1 Z S2000 ← เรียก Tool 1 แกน Z ความเร็ว 2000 RPM

TOOL DEF พารามิเตอร์

Paramความหมาย
[N]หมายเลข Tool (1–32767)
L+0Tool length offset (คงที่ที่ +0)
R+[n]Tool radius (mm) — ต้องตรงจริง

TOOL CALL พารามิเตอร์

Paramความหมาย
[N]หมายเลข Tool ที่ต้องการ
Zแกน Spindle (X, Y, Z, U, V, W)
S[rpm]ความเร็วรอบ Spindle
F[feed]Feed rate (optional)
DR+[n]Delta radius (ปรับ offset เล็กน้อย)

ขนาดดอกเจาะ (R → Ø)

  • R+2.25 → Ø4.5mm
  • R+2.5 → Ø5mm
  • R+3 → Ø6mm
  • R+3.5 → Ø7mm
  • R+3.75 → Ø7.5mm
  • R+4 → Ø8mm
  • R+5 → Ø10mm
TOOL CALL 0 Z: เรียก Tool 0 หมายถึง "ไม่มี Tool" — ใช้ก่อน M6 เพื่อปล่อย Spindle ก่อนเปลี่ยนดอก

4. ลำดับ Start-up (ตายตัว 3 บรรทัด)

Start-up Sequence — เหมือนกันทุกไฟล์ ห้ามเปลี่ยน
5L Z+50 R0 F10000 M3 ← ยก Z safe, ยกเลิก comp, เปิด Spindle CW
6L X-2 Y-2 F10000 ← ไปตำแหน่งนอกชิ้นงาน (Spindle เสถียร)
7L Z+50 F1000 M8 ← ยืนยัน Z+50, เปิด Coolant

การนับบรรทัด

ช่วงลำดับหมายเหตุ
0–7+1Header + Tool + Start-up
10 เป็นต้นไป+2กระโดดข้าม 8, 9 เสมอ
FMAX: สามารถใช้ FMAX แทน F10000 ได้ในเครื่อง TNC รุ่นใหม่ — หมายถึง Rapid traverse สูงสุดของเครื่อง

5. คำสั่งการเคลื่อนที่ (L Command)

L คือ Linear move เทียบเท่า G01 ใน Fanuc — รองรับทั้ง Absolute และ Incremental

5.1 รูปแบบคำสั่ง L

ไวยากรณ์ L Command
—L X±value Y±value Z±value R[comp] F[feed] M[code]
ส่วนประกอบความหมายตัวอย่าง
X±n Y±n Z±nพิกัด Absolute (ค่าจริง)X-312.000 Y0.000
IX±n IY±n IZ±nIncremental (เพิ่มจากตำแหน่งปัจจุบัน)IX+50 IY-10
R0 / RL / RRRadius compensation: ยกเลิก / ซ้าย / ขวาRL F500
F[n]Feed rate — ถ้าไม่ใส่จะใช้ค่าเดิมF1000 หรือ FMAX
M[n]M-code ที่ทำงานพร้อมกันM3

6. Radius Compensation (R0 / RL / RR)

Heidenhain ใช้ RL และ RR แทน G41/G42 ของ Fanuc เพื่อชดเชยรัศมีดอกเมื่อกัดตาม Contour

CodeFanuc เทียบความหมายใช้เมื่อ
R0G40ยกเลิก compensationการเคลื่อนที่ทั่วไป, เริ่ม/จบโปรแกรม
RLG41ชดเชยรัศมีทางซ้ายของเส้น contourกัด Outside contour CCW / Inside CW
RRG42ชดเชยรัศมีทางขวากัด Outside contour CW / Inside CCW
ตัวอย่างการใช้ RL สำหรับกัด Contour
10L X+50 Y+75 RL F500 ← เริ่ม compensation ซ้าย (เข้า contour)
12L X+100 Y+75 ← กัดไปตาม contour
14L X+100 Y+25
16L X+50 Y+25
18L X+50 Y+50 R0 ← ยกเลิก compensation
การเจาะรู (ไฟล์ .hnc ที่วิเคราะห์): ไม่ใช้ RL/RR เลย ใช้แค่ R0 เพราะการเจาะตรง (plunge) ไม่ต้องการ radius comp

Approach & Depart Contour

Codeความหมาย
G26 R[n]Tangential approach เข้า contour ด้วยรัศมี n
G27 R[n]Tangential depart ออกจาก contour

7. การเคลื่อนที่แบบวงกลม (Arc / Circle)

Heidenhain ใช้ CC กำหนดจุดศูนย์กลาง และ C สำหรับ Arc — ต่างจาก G02/G03 ของ Fanuc

ตัวอย่าง Arc Programming CC + C
10CC X+50 Y+50 ← กำหนดจุดศูนย์กลางที่ X50 Y50
12L X+50 Y+75 RL F500 ← เคลื่อนไปจุดเริ่ม arc
14C X+75 Y+50 DR+ ← arc ไป X75 Y50 ทิศ CW (DR+ = CW)
16C X+50 Y+25 DR+
18C X+25 Y+50 DR+
20C X+50 Y+75 DR+ ← ครบวงกลม
คำสั่งความหมายFanuc เทียบ
CC X.. Y..กำหนดจุดศูนย์กลางวงกลม (Circle Center)ไม่มีตรงๆ (ใช้ I, J ใน G02/G03)
C X.. Y.. DR+Circular arc ทิศ CW (Clockwise)G02
C X.. Y.. DR-Circular arc ทิศ CCW (Counter-Clockwise)G03
CR X.. Y.. R±nArc โดยระบุ Radius โดยตรง (ไม่ต้องใช้ CC)G02/G03 + R

Chamfer และ Corner Rounding

คำสั่งความหมาย
CHF [n]ตัดมุม Chamfer ยาว n mm (ใส่ใน L block)
RND R[n]Round มุมด้วยรัศมี n mm (ใส่ใน L block)
Chamfer & Rounding ตัวอย่าง
10L X+0 Y+0 RL F500
12L X+100 CHF 5 ← ตัดมุม 5mm ที่จุดเปลี่ยนทิศถัดไป
14L Y+80 RND R8 ← Round มุม R8 ที่จุดถัดไป
16L X+0

8. Pattern การเจาะ (Manual Peck Drilling)

ไฟล์ .hnc ที่วิเคราะห์ใช้การเจาะแบบ Manual (L commands) แทน CYCL DEF — มี 2 แบบ

8.1 Single Peck (ตื้น <5mm)

Single Peck — ลึก -3mm
10L X-312.000 Y0.000 F1000 ← ไปตำแหน่งรู XY
12L Z30.000 F ← Clearance Z30
14L Z3.000 F ← Approach Z+3
16L Z-3.000 F80 ← เจาะ F80 (Cutting feed)
18L Z30.000 F ← Retract → รูถัดไป

8.2 Multi Peck (ลึก >5mm) — 5mm/Peck

Multi Peck — ลึก 28mm pattern ซ้ำ: peck→Z3→คืน→peck ต่อ
10L X[X] Y[Y] F1000
12L Z10.000 F ← Clearance
14L Z3.000 F ← Approach
16L Z-2.000 F80 ← Peck 1 (ใส่ F ทุกครั้งที่เป็นตัวแรก)
18L Z3.000 F ← Retract Z+3
20L Z-2.000 F ← กลับ -2mm (fast)
22L Z-7.000 F ← Peck 2: -7mm
24L Z3.000 F ← Retract
26L Z-7.000 F
28L Z-12.000 F ← Peck 3: -12mm
... ← ทำซ้ำ pattern (Z3 → คืนลง → -5mm ต่อ) จนถึง Final depth
52L Z-28.000 F ← Final depth
54L Z10.000 F ← Retract → รูถัดไป
สูตร Z sequence: -2 → +3 → -2 → -7 → +3 → -7 → -12 → +3 → -12 → -17 ... (+5mm/peck)

10. M-Code รายการเต็ม ข้อมูลเพิ่มเติม

Heidenhain TNC 640/530/430/426/320 รองรับ M-Code ดังนี้ (ที่ใช้บ่อยในไฟล์ตัวอย่างคือ M2, M3, M8)

M-Code พื้นฐาน

M-Codeความหมายหมายเหตุ
M0หยุดโปรแกรม (Spindle STOP, Coolant OFF)กด Start เพื่อดำเนินต่อ
M1Optional STOP (หยุดเฉพาะเมื่อเปิด opt. stop)
M2จบโปรแกรม (Spindle STOP, Coolant OFF, กลับ block 1)ใช้ในทุกไฟล์
M3เปิด Spindle หมุน CWใช้ที่บรรทัด 5
M4เปิด Spindle หมุน CCW
M5หยุด Spindle
M6เปลี่ยน Tool (Spindle STOP)ต้องใช้ TOOL CALL ก่อน
M8เปิด Coolantใช้ที่บรรทัด 7
M9ปิด Coolant
M13Spindle CW + Coolant ON พร้อมกันแทน M3+M8
M14Spindle CCW + Coolant ON พร้อมกัน
M30เหมือน M2

M-Code ขั้นสูง

M-Codeความหมาย
M89Cycle call แบบ Modal (ทำงานทุกครั้งที่มี L command)
M91พิกัดในบรรทัดนั้นอ้างอิง Machine datum
M92พิกัดอ้างอิงตำแหน่งที่ Machine maker กำหนด (เช่น tool change pos)
M97กัด contour step เล็กๆ
M99Cycle call แบบ Non-modal (ทำครั้งเดียว)
M103ลด feed rate ขณะ plunge ลงตาม factor %
M109/M110Constant contouring speed ที่ cutting edge
M118Superimpose handwheel ระหว่าง program run
M120Look-ahead สำหรับ radius compensation
M128TCPM — รักษาตำแหน่ง tool tip เมื่อหมุนแกน
M140Retract ออกจาก contour ในทิศแกน tool
M148Retract tool อัตโนมัติเมื่อ NC stop

12. Labels & Subroutines (LBL) ข้อมูลเพิ่มเติม

ใช้ LBL สร้าง Subroutine/Label และ CALL LBL เพื่อเรียกซ้ำ — ลดการเขียน code ซ้ำๆ

Subroutine ด้วย LBL ตัวอย่าง
10L X+50 Y+30 FMAX ← ไปรูแรก
12CALL LBL 1 ← เรียก subroutine เจาะ
14L X+100 Y+30 FMAX ← ไปรูสอง
16CALL LBL 1
18L Z+50 FMAX M2
20LBL 1 ← จุดเริ่มต้น Subroutine
22L Z3.000 FMAX
24L Z-10.000 F80
26L Z30.000 FMAX
28LBL 0 ← จุดสิ้นสุด Subroutine (LBL 0 = END)

CALL LBL แบบวนซ้ำ (REP)

Loop ด้วย REP
10CALL LBL 5 REP 4 ← เรียก LBL 5 ซ้ำ 4 ครั้ง
ประโยชน์: ถ้ามีรูที่มี pattern เจาะเหมือนกัน 10 รู ให้เขียน Subroutine เจาะ 1 ชุด แล้ว CALL LBL 10 ครั้ง แทนการเขียนซ้ำ 10 ชุด
